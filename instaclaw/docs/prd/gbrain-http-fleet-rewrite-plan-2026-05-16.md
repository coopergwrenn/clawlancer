# gbrain HTTP-sidecar fleet rewrite — implementation plan

**Status**: PLAN ONLY — no code changes yet. Submit to Cooper for review before implementation.
**Date**: 2026-05-16
**Predecessor**: `docs/prd/gbrain-fleet-rollout-2026-05-12.md` (this PRD supersedes the stdio architecture but preserves the harness + dependency chain).
**Sidecar architecture proven on vm-050**: 2026-05-15 canary. Two end-to-end write + read tests succeeded same-minute, 564ms tool latency, no cold-start.

## Scope

Replace the gbrain stdio MCP architecture (per-session spawn, 90+ second cold-start, hallucination/race risk) with a persistent HTTP sidecar (systemd user service, loopback-only, bearer auth) — fleet-wide. Affects:

- 6 existing edge_city VMs (test agents; Cooper confirmed wipes are OK)
- ~144 non-edge_city VMs (untouched; stepGbrain still partner-gated, they never had gbrain)
- May 23 snapshot bake (new VMs come up with HTTP architecture baked in)
- Esmeralda new-attendee VMs (50-500+, all hit HTTP architecture out-of-snapshot)

## New invariants the rewrite must preserve

1. **Parser markers**: `_install-gbrain-on-vm.ts:203-205` matches `PHASE_[A-H]_OK`, `FATAL_*`, `INSTALL_COMPLETE`, `ALREADY_INSTALLED`. The new bash script must emit the same markers in the same positions.
2. **stepGbrain output parsing**: `vm-reconcile.ts:1645-1648` matches `ALREADY_INSTALLED`, `INSTALL_COMPLETE`, `FATAL_*`. Same constraint.
3. **RESULT_OK / RESULT_FAIL contract**: Phase H's verify script emits `RESULT_OK <kvpairs>` or `RESULT_FAIL <code>`. install-gbrain.sh greps for this. Both new scripts must preserve.
4. **Idempotency**: re-running on a fully-installed VM must be ~2s and emit `ALREADY_INSTALLED`, not re-install.
5. **Hot-reload preservation**: mcp.servers.gbrain changes must go through `openclaw mcp set gbrain` (or jq + atomic mv to preserve atomic-write semantics) so the gateway picks them up without a restart.
6. **Rule 5 (verify gateway health after config changes)**: Phase G must `curl localhost:18789/health` and FATAL if not 200.
7. **Rule 10 (verify every config set)**: Phase G must `openclaw mcp show gbrain` after the set and confirm the transport key matches expected.
8. **Rule 22/30 (data preservation)**: Wipe authorized for test agents only (Cooper's correction). The install script defaults to wipe-on-reinstall but exits ALREADY_INSTALLED before any wipe if idempotency check passes. Wipe only fires on a fresh install or detected stale state.
9. **Rule 31 (test failure modes)**: every FATAL_* exit code maps to a specific failure mode the script catches before installing.
10. **Rule 32 (config set ≠ runtime applied)**: Phase G verifies hot-reload via journal grep for `[reload] config hot reload applied (mcp.servers.gbrain)`, not just by re-reading the file.
11. **Rule 34 (DB ↔ disk verify)**: not directly applicable (no DB column tracks gbrain transport state today), but the install must leave disk + DB in a consistent state.
12. **Rule 35 (HTTP sidecar architecture)**: the rule I authored yesterday. This rewrite implements it.

//   V = gbrain version installed
//   T = mcp.servers.gbrain.transport in openclaw.json (must be "streamable-http")

  'export PATH="$HOME/.bun/bin:$PATH"',

  'P=$(ss -lnpt 2>/dev/null | grep -c "127.0.0.1:3131" || echo 0)',

const transportOk = checkMatch?.[2] === "streamable-http";
const serviceOk = checkMatch?.[3] === "active";
const portOk = checkMatch?.[4] === "1";

## install-gbrain.sh rewrite (scripts/install-gbrain.sh)

The bash script keeps the 8-phase structure and the marker contract, but Phases C–H change substantially to install HTTP architecture. **Phase E expands** to include token-mint + systemd-unit-deploy (which used to not exist). **Phase F is replaced** entirely. **Phase G changes** from stdio-shape `openclaw mcp set` to streamable-http-shape. **Phase H replaces** the stdio JSON-RPC harness with HTTP curl.

### Phase A — pre-flight (PRESERVED nearly verbatim)
- Workspace backup (`~/.openclaw/session-backups/$TS-pre-gbrain.tar.gz`)
- openclaw.json backup
- openclaw binary present
- OPENAI_API_KEY in `~/.openclaw/.env` (≥20 chars)
- GBRAIN_ANTHROPIC_API_KEY in `~/.openclaw/.env` (≥20 chars)
- **NEW: disk ≥10GB free** (per PRD §5.3 — pre-flight disk check)
- Idempotency check: V + T + S + P (same four invariants as stepGbrain) → exit 0 with `ALREADY_INSTALLED` if all match

Exit codes: 1 (backup), 2 (prereqs), **NEW: 1a for disk**

### Phase B — install Bun (PRESERVED verbatim)
- Install unzip if missing
- Install Bun via curl|bash
- Verify `bun --version`
- Exit code 3 (BUN_INSTALL_FAILED)

### Phase C — clone + checkout pinned commit (NEW: install path correction)
- **Path change**: clone to `~/gbrain` (canonical per Garry's `INSTALL_FOR_AGENTS.md`), not `~/.bun/install/global/node_modules/gbrain` (which is the bun-global path; vm-050 was set up at this path historically but the canonical path is `~/gbrain`)
- **Version target**: `git checkout baf1a47` (v0.35.0.0)
- **Local-modification handling**: vm-050's gbrain had a chmod-only mode change to src/cli.ts; the new script does `git checkout -- .` before merge to discard any local mods
- Verify HEAD matches pinned commit; on drift: FATAL_CHECKOUT_DRIFT, rm -rf, exit 5
- Exit codes 4, 5 (CLONE_FAILED, CHECKOUT_DRIFT)

### Phase D — bun install + bun link (PRESERVED with version-regex update)
- `cd ~/gbrain && bun install` (5min timeout)
- `bun link` (creates symlink ~/.bun/bin/gbrain → ~/gbrain/src/cli.ts)
- Verify `gbrain --version` matches `GBRAIN_PINNED_VERSION` using **`[0-9][0-9.]+` regex** (accepts 4-segment versions like 0.35.0.0)
- Exit codes 6, 7, 8 (BUN_INSTALL_FAILED, BUN_LINK_FAILED, VERSION_MISMATCH)

### Phase E — fresh PGLite init + mint bearer token + write systemd unit (REWRITTEN)
- **E1**: Stop gbrain.service if it exists (release PGLite lock); pkill any orphan `bun run.*gbrain` processes
- **E2**: WIPE existing PGLite if present: tar.gz backup to ~/.gbrain/brain.pglite.PRE-WIPE-$TS.tar.gz, then `rm -rf ~/.gbrain/brain.pglite ~/.gbrain/config.json`
- **E3**: `unset GBRAIN_DATABASE_URL DATABASE_URL; cd ~/gbrain && gbrain init --pglite` (v0.35.0.0 reads engine from config.json, rejects pglite:// URLs in env vars)
- **E4**: Verify ~/.gbrain/brain.pglite/ exists + config.json contains `"engine":"pglite"`
- **E5**: Mint bearer token via PGLite-direct INSERT (gbrain auth create broken on PGLite — Known Issue #1 from May-15 PRD; uses inline `bun run` heredoc with @electric-sql/pglite, deletes any prior `openclaw-vm` row, INSERTs new row, writes plaintext to `~/.gbrain/openclaw-bearer-token.txt` mode 600, verifies hash match)
- **E6**: Render systemd unit (Environment= for OPENAI_API_KEY + ANTHROPIC_API_KEY pulled from .env, plus PATH + HOME + GBRAIN_EMBEDDING_MODEL), write to `~/.config/systemd/user/gbrain.service` mode 644
- **E7**: `systemctl --user daemon-reload && systemctl --user enable --now gbrain.service`
- **E8**: Poll for `is-active=active AND port 3131 bound to 127.0.0.1 AND /health=200` for up to 30s; FATAL with journalctl tail on failure
- Exit codes 9-15:
  - 9: PGLITE_INIT_FAILED
  - 10: TOKEN_MINT_FAILED
  - 11: TOKEN_HASH_MISMATCH (file vs DB)
  - 12: SYSTEMD_UNIT_WRITE_FAILED
  - 13: SIDECAR_START_FAILED
  - 14: SIDECAR_HEALTH_FAILED (active but /health != 200)
  - 15: SIDECAR_BOUND_PUBLIC (port on 0.0.0.0 — security invariant)

### Phase F — HTTP smoke test (REPLACES the stdio probe)
- `curl localhost:3131/health` returns 200 with `{"status":"ok","engine":"pglite"}`
- `curl -H 'Authorization: Bearer $TOKEN' /mcp` JSON-RPC initialize returns protocolVersion + serverInfo.name=gbrain
- `curl /mcp` tools/list returns ≥5 expected tools (put_page, get_page, search, recall, query)
- External-IP REFUSED test: `bash -c '</dev/tcp/$EXT_IP/3131'` must fail (refused)
- Exit code 16 (HTTP_SMOKE_FAILED) — diagnostic includes which sub-check failed

### Phase G — wire MCP via openclaw mcp set + verify hot-reload (REWRITTEN for streamable-http shape)
- Build streamable-http JSON config inline via python heredoc:
  ```json
  {
    "transport": "streamable-http",
    "url": "http://127.0.0.1:3131/mcp",
    "headers": {"Authorization": "Bearer <token>"},
    "connectionTimeoutMs": 5000
  }
  ```
- `openclaw mcp unset gbrain` first (defensive — handles stdio→HTTP transition cleanly, avoids merge ambiguity)
- `openclaw mcp set gbrain "$(cat $GBRAIN_JSON_FILE)"` — exit-code check, fail FATAL_MCP_SET_FAILED on non-zero
- Sleep 2 (hot-reload window per existing pattern)
- **NEW: verify-after-set per Rule 10 + Rule 32**: `openclaw mcp show gbrain | grep -q 'streamable-http'` — fail FATAL_VERIFY_AFTER_SET_FAILED if missing
- **NEW: empirical hot-reload confirm per Rule 32**: `journalctl --user -u openclaw-gateway --since '5 seconds ago' | grep -q 'config hot reload applied'` — if missing, log a WARN (not FATAL; the file write succeeded, the journal grep is best-effort because timestamp resolution can drift)
- Gateway health probe per Rule 5: `curl localhost:18789/health` returns 200 → FATAL_GATEWAY_UNHEALTHY_POST_HOT_RELOAD on non-200
- Exit codes 17 (MCP_SET_FAILED), 18 (VERIFY_AFTER_SET_FAILED), 19 (GATEWAY_UNHEALTHY_POST_HOT_RELOAD)
- **Rollback on failure**: `openclaw mcp unset gbrain` before exit

### Phase H — real round-trip via HTTP MCP (REPLACES stdio harness)
- Upload `verify-gbrain-mcp.py` (HTTP version, see below) via the existing TS-wrapper pattern at `/tmp/verify-gbrain-mcp.py`
- Run with required env: `OPENAI_API_KEY`, `ANTHROPIC_API_KEY` (sourced from .env), `GBRAIN_BEARER_TOKEN` (sourced from token file), `MARKER_TS` (for diagnostic correlation)
- Timeout 180s
- Parse `RESULT_OK <kvpairs>` or `RESULT_FAIL <code>` from stdout
- On `RESULT_FAIL` or non-zero exit: `openclaw mcp unset gbrain` rollback, exit 20 (VERIFY_GATE_FAILED)

## verify-gbrain-mcp.py rewrite (scripts/verify-gbrain-mcp.py)

Replace the stdio JSON-RPC subprocess harness with HTTP-curl-style requests via the `urllib` stdlib. **No more subprocess management.** The verify is simpler:

```python
#!/usr/bin/env python3
# verify-gbrain-mcp.py (HTTP sidecar verify gate, Rule 35)
#
# Replaces the stdio subprocess harness. Drives put_page → get_page (or query)
# round-trip via HTTP requests to the running gbrain.service sidecar.

# Required env:
#   GBRAIN_BEARER_TOKEN — from ~/.gbrain/openclaw-bearer-token.txt
#   MARKER_TS — for stable correlation across runs

# Output contract (UNCHANGED from stdio version):
#   RESULT_OK <kvpairs> on success
#   RESULT_FAIL <code> on any failure

import json, os, sys, time, urllib.request, urllib.error

URL = "http://127.0.0.1:3131/mcp"
HEALTH_URL = "http://127.0.0.1:3131/health"
TIMEOUT = 30  # per-request

def fail(code, **kw):
    kv = " ".join(f"{k}={v}" for k, v in kw.items())
    print(f"RESULT_FAIL {code} {kv}")
    sys.exit(1)

def post_mcp(token, payload):
    """Send JSON-RPC over HTTP with bearer auth. Returns parsed result dict or raises."""
    req = urllib.request.Request(
        URL,
        method="POST",
        headers={
            "Content-Type": "application/json",
            "Accept": "application/json, text/event-stream",
            "Authorization": f"Bearer {token}",
        },
        data=json.dumps(payload).encode(),
    )
    with urllib.request.urlopen(req, timeout=TIMEOUT) as resp:
        body = resp.read().decode()
        # Server-Sent Events stream — single "data: {...}" line
        for line in body.splitlines():
            if line.startswith("data: "):
                return json.loads(line[6:])
        # Plain JSON fallback
        return json.loads(body)

def main():
    token = os.environ.get("GBRAIN_BEARER_TOKEN", "").strip()
    if not token:
        fail("NO_TOKEN", hint="set GBRAIN_BEARER_TOKEN from ~/.gbrain/openclaw-bearer-token.txt")
    marker_ts = os.environ.get("MARKER_TS", str(int(time.time())))

    # 1. /health (no auth) — confirms sidecar is up
    try:
        with urllib.request.urlopen(HEALTH_URL, timeout=5) as r:
            health = json.loads(r.read())
            if health.get("status") != "ok":
                fail("HEALTH_NOT_OK", health=health.get("status"))
    except Exception as e:
        fail("HEALTH_UNREACHABLE", err=str(e)[:120])

    # 2. /mcp initialize
    try:
        init = post_mcp(token, {
            "jsonrpc": "2.0", "id": 1, "method": "initialize",
            "params": {
                "protocolVersion": "2025-06-18",
                "capabilities": {},
                "clientInfo": {"name": "verify-gbrain-mcp", "version": "1.0"},
            },
        })
        srv = init.get("result", {}).get("serverInfo", {})
        if srv.get("name") != "gbrain":
            fail("INIT_UNEXPECTED_SERVER", got=srv.get("name"))
        server_version = srv.get("version", "unknown")
    except urllib.error.HTTPError as e:
        if e.code == 401:
            fail("AUTH_401", hint="bearer token rejected — file/DB hash mismatch?")
        fail("INIT_HTTP_ERROR", code=e.code)
    except Exception as e:
        fail("INIT_ERROR", err=str(e)[:120])

    # 3. tools/list — discover put + retrieve tool names defensively
    tools = post_mcp(token, {"jsonrpc": "2.0", "id": 2, "method": "tools/list", "params": {}})
    tool_names = [t.get("name") for t in tools.get("result", {}).get("tools", [])]
    if "put_page" not in tool_names:
        fail("NO_PUT_PAGE", tool_count=len(tool_names))
    retrieve_tool = "get_page" if "get_page" in tool_names else (
        "search" if "search" in tool_names else None
    )
    if not retrieve_tool:
        fail("NO_RETRIEVE_TOOL", tool_count=len(tool_names))

    # 4. tools/call put_page — write a marker page
    marker_slug = "_gbrain-install-verify"
    marker_body = f"gbrain HTTP sidecar verify — marker_ts={marker_ts}"
    put_result = post_mcp(token, {
        "jsonrpc": "2.0", "id": 3, "method": "tools/call",
        "params": {
            "name": "put_page",
            "arguments": {"slug": marker_slug, "content": marker_body},
        },
    })
    put_content = put_result.get("result", {}).get("content", [{}])[0].get("text", "")
    if put_result.get("result", {}).get("isError"):
        fail("PUT_ISERROR", err=put_content[:200])
    if "created_or_updated" not in put_content and "created" not in put_content:
        fail("PUT_UNEXPECTED", body=put_content[:200])

    # 5. tools/call get_page (or search) — read it back
    if retrieve_tool == "get_page":
        get_args = {"slug": marker_slug}
    else:
        # search needs a query that matches the marker_ts
        get_args = {"query": marker_ts}
    get_result = post_mcp(token, {
        "jsonrpc": "2.0", "id": 4, "method": "tools/call",
        "params": {"name": retrieve_tool, "arguments": get_args},
    })
    get_content = get_result.get("result", {}).get("content", [{}])[0].get("text", "")
    if get_result.get("result", {}).get("isError"):
        fail("RETRIEVE_ISERROR", err=get_content[:200])
    if marker_ts not in get_content:
        fail("MARKER_NOT_FOUND", retrieve_tool=retrieve_tool, body=get_content[:200])

    print(
        f"RESULT_OK marker_ts={marker_ts} "
        f"put_tool=put_page retrieve_tool={retrieve_tool} "
        f"server_version={server_version} tools_count={len(tool_names)}"
    )

if __name__ == "__main__":
    main()
```

**Failure-mode coverage (Rule 31)**:
- `NO_TOKEN` — caller forgot to source the token file
- `HEALTH_UNREACHABLE` — sidecar not running (caught at /health, before any auth)
- `HEALTH_NOT_OK` — sidecar up but PGLite broken
- `AUTH_401` — bearer mismatch (file vs DB SHA256 hash differ)
- `INIT_UNEXPECTED_SERVER` — wrong MCP server bound to port 3131
- `NO_PUT_PAGE` — gbrain version doesn't expose put_page
- `PUT_ISERROR` — write failed (PGLite write error, OpenAI embedding fail, schema mismatch — the THREE bug classes the old Phase F missed)
- `PUT_UNEXPECTED` — response shape changed (gbrain version upgrade caught)
- `RETRIEVE_ISERROR` — read failed
- `MARKER_NOT_FOUND` — written but unreadable (most likely embed failure → semantic search miss; or PGLite write didn't commit)

## Rollback plan

If anything goes wrong with the rewrite:

### If the rewrite hasn't shipped yet (still in PR / pre-merge)
- Discard the branch; existing code in main stays at stdio v0.28.1

### If shipped but stepGbrain is broken
- Set `GBRAIN_INSTALL_ENABLED=false` in Vercel env (1 line in dashboard)
- stepGbrain returns immediately on Gate 3
- Fleet stays at whatever state it's in (stdio or HTTP) until the rule's re-enabled
- Time-to-mitigate: <60s

### If shipped and a VM gets stuck in a partial-install state
- The install script's FATAL paths leave the VM in a recoverable state:
  - Phase A-D failures: nothing changed (or `rm -rf ~/gbrain` reverts the clone). Re-run.
  - Phase E failures (PGLite, token, systemd): partial state exists. Re-run is safe (Phase E1 cleans up).
  - Phase F-H failures: `openclaw mcp unset gbrain` rollback fires automatically before exit; openclaw.json back to pre-flip state.

### If shipped and the reconciler itself crashes on the new code
- Vercel rolls back to prior deploy via git revert + redeploy (5 min)
- All assigned VMs continue serving via their last-known-good cv state

## Failure-mode for the rewrite itself

The reconciler runs every 3 min. If I ship broken code:
- Wrong idempotency check → wrong install fires on every cycle for 6 VMs = constant churn
- Wrong install script → 6 VMs broken simultaneously, all visible in `instaclaw_strict_holds`
- Wrong constants → version-mismatch fatal on every install attempt

**Mitigation**: GBRAIN_INSTALL_ENABLED env flag is the kill switch. Cooper or any operator can flip it `false` in Vercel within 60s to halt all installs.
