## Glossary

- **Static-extracted**: skill lives in `instaclaw/skills/<name>/`, deployed at configure-time by `lib/ssh.ts` (via `fs.readFileSync` on the local repo) and/or by `lib/vm-reconcile.ts:stepSkills` (universal SKILL.md + `scripts/*.sh`/`*.py` loop).
- **Inlined**: contents embedded as TypeScript template-literal constants. No runtime file system reads. Immune to NFT tracer silently dropping files.
- **NFT-RISK**: relies on `outputFileTracingIncludes: ["./skills/**/*"]` to ship files into Vercel bundle. Confirmed broken for `.sh` (the dispatch bug). Suspected broken for `.tsx`/`.py` in some cases.
- **Universal**: every assigned user gets this skill regardless of partner/tier/flags. The catch block does `recordFailure("skill_X", err, false)` (3rd arg `false` = non-fatal — install failure does NOT block configure completion, except for `dispatch_deploy` which is `critical=true`).
- **Conditional**: gated by `config.partner === "..."` OR DB flag (e.g., `agdp_enabled`).

## Risk Summary for Cloud-Init Migration

If snapshot is baked with all skills pre-staged, the NFT-RISK concern disappears entirely (no runtime `fs.readFileSync` from `skills/`). The risks DURING bake (when files have to come from somewhere):

| Risk | Mitigation |
|---|---|
| Snapshot misses `.sh`/`.py` due to NFT during bake-script generation | Bake script should pull from local repo (`git clone`), not from Vercel function. Verified bake script `scripts/_bake-snapshot-v62.ts` uses local extraction. |
| Skills update after snapshot baked | Reconciler `stepSkills` + per-skill cron auto-pulls (bankr, consensus-2026, edge-esmeralda). For static skills, manifest version bump → reconciler re-deploys SKILL.md from local repo. |
| Partner-conditional content (edge-esmeralda) baked into universal snapshot | DON'T bake. Skill is git-cloned at configure-time per-VM only if `partner=edge_city`. Per Rule 9. |
| Sibling-clone (dgclaw at `$HOME`) baked into snapshot | Acceptable for universal install (every VM gets the SKILL.md). The CLI tools install only if `agdp_enabled=true`; baking the static SKILL.md is harmless. |
| New skill added post-bake | Reconciler `stepSkills` picks it up automatically on next tick. Bake snapshots accumulate skills over manifest versions. |
